题目大意

题目归纳

给定一个数组大小 nnn (1≤n≤1041 \le n \le 10^41≤n≤104)，需要构建并按特定格式可视化一个线段树。

线段树构建规则

  • 根节点：对应半开区间 [0,n)[0, n)[0,n)。

  • 分裂规则：对于非叶子节点 [L,R)[L, R)[L,R)，其中点为 M=⌊(L+R)/2⌋M = \lfloor (L+R)/2 \rfloorM=⌊(L+R)/2⌋。左子节点为 [L,M)[L, M)[L,M)，右子节点为 [M,R)[M, R)[M,R)。

  • 叶子节点：对应区间长度为 1 的节点 [i,i+1)[i, i+1)[i,i+1)。

可视化输出规则

  • 分层输出：树的根在第一行，子节点在下一行，以此类推。

  • 节点格式：显示为 [L;R)。

  • 分隔竖线：如果节点 [L,R)[L, R)[L,R) 不是叶子，必须在其分号 ; 的正下方（即同一列），从该节点的下一行开始直到整个图形的最后一行，绘制竖线 |。这表示左右子树的物理分隔。

  • 排版约束：

    • 同一行内的元素（节点文本或竖线）之间至少保留 1 个空格。
    • 在满足对齐条件的前提下，空格数量应尽可能少。
    • 整个图形靠左对齐（即至少有一行的行首没有空格）。
    • 行末不得有多余空格。

样例展示与解释

解释：

  • 第 1 行：根节点 [0,3)[0, 3)[0,3)。其中点为 1，分号 ; 所在列决定了后续行的主分隔线位置。

  • 第 2 行：

    • 左子节点 [0,1)[0, 1)[0,1)。
    • 中间是根节点 [0,3)[0, 3)[0,3) 延伸下来的竖线 |。
    • 右子节点 [1,3)[1, 3)[1,3)。

  • 第 3 行：

    • 左侧空位（因为 [0,1)[0, 1)[0,1) 是叶子）。
    • 根节点的竖线 | 继续延伸。
    • [1,3)[1, 3)[1,3) 分裂为 [1,2)[1, 2)[1,2) 和 [2,3)[2, 3)[2,3)，中间由 [1,3)[1, 3)[1,3) 的分号 ; 延伸出的竖线 | 分隔。

  • 对齐：所有竖线 | 都严格位于其父节点 ; 的正下方，且元素间距最小化。

  • 第 2 行的节点 [0,3)[0, 3)[0,3) 和 [3,7)[3, 7)[3,7) 的竖线分别贯穿了第 3、4 行。

思路讲解

官方题解（写的挺好的）

让我们考虑一个图GGG，其中u↔vu\leftrightarrow vu↔v是一条边，当且仅当fuf_ufu​和fvf_vfv​是独立的。那么，根据问题定义，一个有序子集是图GGG中的一个团（完全子图）。问题是，团问题众所周知是NP完全的。我们能否利用任何其他属性？
给定的函数是连续函数。也就是说，如果对于所有x∈Rx \in \mathbb{R}x∈R，fi(x)≠fj(x)f_i(x) \neq f_j(x)fi​(x)=fj​(x)，则以下任一条件成立：

  • fi(x)>fj(x)f_i(x) \gt f_j(x)fi​(x)>fj​(x)对于所有x∈Rx \in \mathbb{R}x∈R。
  • fi(x)<fj(x)f_i(x) \lt f_j(x)fi​(x)<fj​(x)对于所有x∈Rx \in \mathbb{R}x∈R。
    让我们表示为u≺vu \prec vu≺v，如果对于所有x∈Rx \in \mathbb{R}x∈R，fu(x)<fv(x)f_u(x) \lt f_v(x)fu​(x)<fv​(x)。我们将考虑该关系的独特属性。
  • 关系≺\prec≺是传递的。也就是说，如果a≺ba \prec ba≺b和b≺cb \prec cb≺c，那么a≺ca \prec ca≺c。
  • 如果u≺vu \prec vu≺v，那么显然v≺uv \prec uv≺u是不可能的。
  • 不能存在任何uuu使得u≺uu \prec uu≺u成立。
    这种关系在数学中也被称为严格偏序。
    让我们构造一个有向图DDD，其中u→vu \to vu→v是一条边，当且仅当u≺vu \prec vu≺v。有序子集中的所有索引可以按≺\prec≺排序，因此大小为kkk的有序子集对应于DDD中kkk个顶点的路径。因此，现在我们将问题简化为DDD中的最长路径问题。
    由于≺\prec≺的特性，图DDD不能有循环，这意味着图DDD是一个有向无环图（DAG）。正如我们已经知道的，DAG上的最长路径问题可以使用DP来解决。
    让我们定义两个DP状态。dpl[v]\mathtt{dp}_l[v]dpl​[v]定义为以vvv结尾的最长路径的长度（顶点数），dpr[v]\mathtt{dp}_r[v]dpr​[v]定义为以vvv开始的最长路径的长度。那么包含顶点vvv的最长路径的长度恰好是dpl[v]+dpr[v]−1\mathtt{dp}_l[v]+\mathtt{dp}_r[v]-1dpl​[v]+dpr​[v]−1。我们可以在O(n2)\mathcal{O}(n^2)O(n2)时间内计算出dpr[v]\mathtt{dp}_r[v]dpr​[v]和dpr[v]\mathtt{dp}_r[v]dpr​[v]，问题得以解决。
错误的使用了 bfs 求解最长链

不应该使用这个 BFS，无论你怎么样 BFS 它都不太能够求解这个最长的问题，因为毕竟 BFS 是求最短路径。

那么我们实际上可以使用记忆化的 DFS 去解决这个问题。

实际上来说呢，如果这个图它是一个完全图，它没有任何其他的性质的话，实际上是一个 NP 问题，也就是团问题。因此呢，实际上这个关系之间具有严格偏序关系。因此这个图实际上是一个 DAG。我们是就是要在一个 DAG 上去求包含该节点的一个最长链。

那么其实不难想到，这个是真不难想到，就是你去求包含一个节点的最长链的话，直接去求是比较困难的。你可以求，先求以这个点为起点的最长链（长度），再去求以这个点为终点的最长链（长度）。

不过其实求以这个点为终点的最长链的长度，你可能会说，哎，我不会求啊，这个怎么求？哎，实际上很简单，你就把图所有的边全部反向，然后再用跑，这个求以这个点为起点的最长链长度再跑一遍就好了。

实际上来说，这个东西，这个记忆化 DFS 能跑，也是因为这张图具有 DAG 性质。你实在不放心，也可以用拓扑排序去跑一个 DP 也不难，反正。

**遗漏了这个做差式二次函数退化的情况**，导致这个 Q U A D 的小于号运算符出现了判断错误。
TLE ，注意⚠️，边的数量也会影响记忆化 DFS 的时间复杂度

https://codeforces.com/contest/2195/submission/363305041

其实 TLE 的原因很简单，就是 DFS 实际上是 O(V+E)=O(N2)O(V + E) = O(N^2)O(V+E)=O(N2)，DFS 实际上是 ON 方的。那么我们跑了 N 遍 DFS，跑了 N 遍记忆化 DFS，那么实际上是 O(N3)O(N^3)O(N3)。DFS 还是不要去忽略这个边的影响。
样例全部正确。总结一下这个 TLE 的原因：
你的代码本身确实是记忆化搜索，单次 DFS 是 O(V+E)=O(N2)O(V + E) = O(N^2)O(V+E)=O(N2)。但问题在于你在 for 循环里面每轮都 dp.assign(N+2,0) 和 vis.assign(N+2,0)，把之前所有记忆化结果全部清空了。这导致：

  • 第 1 次从节点 1 出发做 DFS，算完所有可达节点的 dp 值
  • 第 2 次清空，从节点 2 重新算一遍
  • 第 3 次清空，从节点 3 重新算一遍
  • ……共 NNN 次，每次 O(N2)O(N^2)O(N2)，总共 O(N3)O(N^3)O(N3)
    修复只需要把这两行初始化移到循环外面。这样第一个 dfs(dfs,1) 可能就把大部分节点的 dp 都算好了，后续的 dfs(dfs,i) 碰到已访问节点直接返回，总复杂度降为 O(N2)O(N^2)O(N2)。
