题目1（单点修改、区间查询、最大子段和）

题目大意

给定一个长度为 nnn 的数组 aaa，初始元素范围为 −109≤ai≤109-10^9 \le a_i \le 10^9−109≤ai​≤109。你需要处理 mmm 次操作，每次操作将索引 iii 处的元素修改为 vvv。

你需要输出 m+1m+1m+1 行结果：

  1. 在所有操作之前，数组中最大子段和（连续子数组的最大和）。

  2. 在每次修改操作后，当前数组的最大子段和。

注意：

  • 子段可以为空，空子段的和定义为 000。因此答案至少为 000。

  • 1≤n,m≤1000001 \le n, m \le 1000001≤n,m≤100000。

  • 索引 iii 从 000 开始 (0≤i<n0 \le i < n0≤i<n)。

输入格式

第一行包含两个整数 nnn 和 mmm。
第二行包含 nnn 个整数，表示数组的初始状态。
接下来的 mmm 行，每行包含两个整数 iii 和 vvv，表示将 aia_iai​ 修改为 vvv。

输出格式

输出共 m+1m+1m+1 行，每行一个整数，表示当前状态下的最大子段和。

样例 1

样例 1 输出

样例 1 解释

  1. 初始状态：数组为 [5, -4, 4, 3, -5]。
  • 最大子段和为 5+(−4)+4+3=85 + (-4) + 4 + 3 = 85+(−4)+4+3=8（索引 000 到 333）。
  • 输出 8。
  1. 第一次操作 4 3：将 a4a_4a4​ 修改为 333。
  • 数组变为 [5, -4, 4, 3, 3]。
  • 最大子段和为 5+(−4)+4+3+3=115 + (-4) + 4 + 3 + 3 = 115+(−4)+4+3+3=11（索引 000 到 444）。
  • 输出 11。
  1. 第二次操作 3 -1：将 a3a_3a3​ 修改为 −1-1−1。
  • 数组变为 [5, -4, 4, -1, 3]。
  • 最大子段和为 5+(−4)+4+(−1)+3=75 + (-4) + 4 + (-1) + 3 = 75+(−4)+4+(−1)+3=7（索引 000 到 444）。
  • 输出 7。

样例 2 输出

样例 2 解释

  1. 初始状态：数组全为负数 [-2, -1, -5, -4]，最大子段为空子段，和为 0。

  2. 第一次操作 1 3：数组变为 [-2, 3, -5, -4]，最大子段为 [3]，和为 3。

  3. 第二次操作 3 2：数组变为 [-2, 3, -5, 2]，最大子段仍为 [3]，和为 3。

题目一（普通的单点修改、区间和查询线段树）

题目描述

需要编写一个标准的线段树来维护数组的区间和，支持单点修改和区间求和操作。

输入格式

第一行包含两个整数 nnn 和 mmm (1≤n,m≤1000001 \le n, m \le 1000001≤n,m≤100000)，分别表示数组的大小和操作的次数。
第二行包含 nnn 个整数 aia_iai​，表示数组的初始状态 (0≤ai≤1090 \le a_i \le 10^90≤ai​≤109)。
接下来的行包含操作的描述。每个操作的描述如下：

  • 1 i v：将索引为 iii 的元素设置为 vvv (0≤i<n0 \le i < n0≤i<n, 0≤v≤1090 \le v \le 10^90≤v≤109)。

  • 2 l r：计算索引从 lll 到 r−1r-1r−1 的元素之和 (0≤l<r≤n0 \le l < r \le n0≤l<r≤n)。

输出格式

对于每个第二种类型的操作，打印对应的和。

样例输入

样例输出

样例解释

初始数组为 [5, 4, 2, 3, 5]（注意题目中下标从 0 开始）。

  1. 操作 2 0 3：计算下标 [0,3)[0, 3)[0,3) 即 0,1,20, 1, 20,1,2 的和。5+4+2=115 + 4 + 2 = 115+4+2=11。

  2. 操作 1 1 1：将下标为 1 的元素修改为 1。数组变为 [5, 1, 2, 3, 5]。

  3. 操作 2 0 3：再次计算下标 [0,3)[0, 3)[0,3) 的和。5+1+2=85 + 1 + 2 = 85+1+2=8。

  4. 操作 1 3 1：将下标为 3 的元素修改为 1。数组变为 [5, 1, 2, 1, 5]。

  5. 操作 2 0 5：计算下标 [0,5)[0, 5)[0,5) 即整个数组的和。5+1+2+1+5=145 + 1 + 2 + 1 + 5 = 145+1+2+1+5=14。

题目大意

题目归纳

给定一个数组大小 nnn (1≤n≤1041 \le n \le 10^41≤n≤104)，需要构建并按特定格式可视化一个线段树。

线段树构建规则

  • 根节点：对应半开区间 [0,n)[0, n)[0,n)。

  • 分裂规则：对于非叶子节点 [L,R)[L, R)[L,R)，其中点为 M=⌊(L+R)/2⌋M = \lfloor (L+R)/2 \rfloorM=⌊(L+R)/2⌋。左子节点为 [L,M)[L, M)[L,M)，右子节点为 [M,R)[M, R)[M,R)。

  • 叶子节点：对应区间长度为 1 的节点 [i,i+1)[i, i+1)[i,i+1)。

可视化输出规则

  • 分层输出：树的根在第一行，子节点在下一行，以此类推。

  • 节点格式：显示为 [L;R)。

  • 分隔竖线：如果节点 [L,R)[L, R)[L,R) 不是叶子，必须在其分号 ; 的正下方（即同一列），从该节点的下一行开始直到整个图形的最后一行，绘制竖线 |。这表示左右子树的物理分隔。

  • 排版约束：

    • 同一行内的元素（节点文本或竖线）之间至少保留 1 个空格。
    • 在满足对齐条件的前提下，空格数量应尽可能少。
    • 整个图形靠左对齐（即至少有一行的行首没有空格）。
    • 行末不得有多余空格。

样例展示与解释

样例 1 (n=3n=3n=3)

解释：

  • 第 1 行：根节点 [0,3)[0, 3)[0,3)。其中点为 1，分号 ; 所在列决定了后续行的主分隔线位置。

  • 第 2 行：

    • 左子节点 [0,1)[0, 1)[0,1)。
    • 中间是根节点 [0,3)[0, 3)[0,3) 延伸下来的竖线 |。
    • 右子节点 [1,3)[1, 3)[1,3)。

  • 第 3 行：

    • 左侧空位（因为 [0,1)[0, 1)[0,1) 是叶子）。
    • 根节点的竖线 | 继续延伸。
    • [1,3)[1, 3)[1,3) 分裂为 [1,2)[1, 2)[1,2) 和 [2,3)[2, 3)[2,3)，中间由 [1,3)[1, 3)[1,3) 的分号 ; 延伸出的竖线 | 分隔。

  • 对齐：所有竖线 | 都严格位于其父节点 ; 的正下方，且元素间距最小化。

样例 2 (n=7n=7n=7)

解释：

  • 展示了更深层的结构。每一层新产生的分支节点都会在其 ; 下方产生一条新的竖线，一直延伸到底部。

  • 例如，根节点 [0,7)[0, 7)[0,7) 的竖线贯穿了第 2、3、4 行。

  • 第 2 行的节点 [0,3)[0, 3)[0,3) 和 [3,7)[3, 7)[3,7) 的竖线分别贯穿了第 3、4 行。

思路讲解

这道题目主要就是考验你的一个实现的这个能力。比较建议的实现方法呢，就是这个宽度的计算呢，先使用一个函数进行总的宽度的计算。然后需要注意啊，这道题目不允许出现后导空格，就是它不会自动忽略后导空格。所以你需要在输出的时候把所有的后导空格给 pop back 给 pop 掉。

https://codeforces.com/gym/106169/submission/363482617 因为后导空格 WA 的一发。

然后还有一个点呢，就是这道题目它使用的是左臂右开，0 base 的，那么你最好也仿照它使用左臂右开0 base 的，不要把它，不要用1 base 的做这道题目，1 base 的做这道题目。是会有问题的，因为因为这个嗯，因为一加一个数，然后再 向下整除二，和0加一个数再向下整除二，它其实这个区间是会有不同的。
