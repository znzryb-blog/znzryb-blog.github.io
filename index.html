题目大意

题目大意

有 2n2n2n 张卡片排成一排，卡片上分别写着数字 1,1,2,2,…,n,n1, 1, 2, 2, \dots, n, n1,1,2,2,…,n,n，初始时全部背面朝上。

你将使用以下“贪心”策略来进行翻牌游戏，每回合精确地翻开两张卡片：

  1. 如果你之前已经翻开过（且未被消除）两张数字相同的卡片，则直接翻开并消除它们。

  2. 否则，翻开从左到右第一张从未被翻开过的卡片，设其数字为 xxx。

  3. 接着，如果之前已经翻开过另一张数字为 xxx 的卡片，则翻开并消除它。

  4. 否则，再次翻开从左到右第一张从未被翻开过的卡片。如果这两张卡片数字相同，则消除；如果不同，则重新翻回背面。

给定 nnn 和 kkk，要求构造一个 111 到 nnn 各出现两次的排列，使得上述算法恰好需要 kkk 个回合才能消除所有卡片。如果存在这样的排列，输出 YES 并输出该排列；否则输出 NO。

输入格式

第一行包含一个整数 ttt（1≤t≤1031 \le t \le 10^31≤t≤103），表示测试数据组数。
接下来每组数据包含一行，包含两个整数 nnn 和 kkk（1≤n≤3×1051 \le n \le 3 \times 10^51≤n≤3×105，1≤k≤1061 \le k \le 10^61≤k≤106）。
保证所有测试数据中 nnn 的总和不超过 3×1053 \times 10^53×105。

输出格式

如果不存在满足条件的卡片排列，输出一行 NO。
如果存在，输出一行 YES，并在下一行输出 2n2n2n 个整数 a1,a2,…,a2na_1, a_2, \dots, a_{2n}a1​,a2​,…,a2n​（1≤ai≤n1 \le a_i \le n1≤ai​≤n），表示卡片上的数字排列。如果有多个合法的排列，输出任意一个即可。

样例解释

对于第一组样例，初始排列为 2 1 2 1，每回合的操作如下：

  • 第 111 回合：翻开第一张未翻过的卡片（数字 222），之前没有记录，接着翻开下一张未翻过的卡片（数字 111）。数字不同，翻回背面。此时记下前两张卡片的数字为 222 和 111。

  • 第 222 回合：翻开第一张未翻过的卡片（第三张，数字 222），由于之前已经翻开过数字为 222 的卡片（第一张），因此翻开第一张，两张卡片相同并消除。

  • 第 444 回合：没有两张已翻开且相同的卡片。翻开第五张卡片（数字 222），由于之前翻开过数字 222（第二张卡片），因此翻开第二张卡片并消除。

  • 第 555 回合：翻开第六张卡片（数字 333），由于之前翻开过数字 333（第三张卡片），因此翻开第三张卡片并消除。
    游戏结束，算法恰好在 k=5k=5k=5 回合赢得游戏。

题目描述
给定 nnn 个不同的声音，每个声音由音量 viv_ivi​ 和音高 pip_ipi​ 两个属性组成。
你需要将这 nnn 个声音排列成一个序列，每个声音恰好出现一次。
一个合法的序列需要同时满足以下两个条件：

  1. 任意两个相邻的声音，必须且仅有一个属性相同（即要么音量相同且音高不同，要么音高相同且音量不同）。

  2. 任意连续的三个声音，它们的音量不能完全相同，音高也不能完全相同（即不能出现连续三个声音音量一样，也不能出现连续三个声音音高一样）。

这道题目的 SPJ 怎么写？

编写这道题 SPJ（Special Judge）的核心难点在于：如何验证“无论怎么走都遇不到”这一条件。

因为题目中描述的移动策略是“任意选择一条出边（arbitrarily traverse）”，这意味着玩家的移动是不确定的。你不能只模拟一条路径，而是必须模拟所有可能的路径。

核心思路：集合模拟（Set Simulation）

要在 SPJ 中解决这个问题，关键点不是记录 Alice 和 Bob “在哪一个点”，而是记录他们当前可能在哪些点的集合。

由于 N≤300N \le 300N≤300，我们可以使用 C++ 的 std::bitset<305> 来高效地维护这个集合。

SPJ 的具体实现步骤

1. 基础校验 (Basic Validation)

首先，必须确保选手输出的图是合法的：

  • 边数对应：选手输出的有向边必须和输入的无向边一一对应（通过 multiset 或排序后比对）。

  • 点范围：点的编号在 1…N1 \dots N1…N 之间。

2. 状态定义 (The Critical Point)

这是 SPJ 最关键的部分。

  • 设 SA(t)S_A^{(t)}SA(t)​ 为第 ttt 轮结束时，Alice 可能所在的所有顶点的集合。

  • 设 SB(t)S_B^{(t)}SB(t)​ 为第 ttt 轮结束时，Bob 可能所在的所有顶点的集合。

初始状态：

SA(0)={Alicestart},SB(0)={Bobstart}S_A^{(0)} = \{Alice_{start}\}, \quad S_B^{(0)} = \{Bob_{start}\}SA(0)​={Alicestart​},SB(0)​={Bobstart​}

3. 状态转移 (Transition)

每一轮模拟时，我们需要计算下一轮的集合 S(t+1)S^{(t+1)}S(t+1)。规则如下：

对于集合中的每一个点 u∈S(t)u \in S^{(t)}u∈S(t)：

  • 情况 A（有出边）：如果 uuu 有出边指向 v1,v2,…v_1, v_2, \dotsv1​,v2​,…，那么在下一轮，Alice/Bob 可能会出现在这些邻居中的任何一个。将 {v1,v2,… }\{v_1, v_2, \dots\}{v1​,v2​,…} 全部加入 S(t+1)S^{(t+1)}S(t+1)。

  • 情况 B（无出边）：如果 uuu 没有出边，根据题目规则，玩家停留在原地。将 uuu 加入 S(t+1)S^{(t+1)}S(t+1)。

4. 碰撞检测 (Collision Check)

在任何时刻 ttt，如果 Alice 和 Bob 的可能位置集合有交集，说明存在一种走法让他们相遇，判为 WA。

  • 判定条件：(S_A & S_B).any() == true

5. 终止条件 (Termination)

模拟不能无限进行下去。我们有两个终止条件：

  1. 相遇：检测到交集，直接 quitf(_wa, ...)****。

  2. **死循环（安全）：**如果当前的“状态对” (SA,SB)(S_A, S_B)(SA​,SB​) 曾经出现过，说明我们进入了一个循环，而且在这个循环中没有发生碰撞。那么继续跑下去也不会碰撞。此时判为 AC。
