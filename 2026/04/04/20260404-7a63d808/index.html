2026 杭电春季联赛 3——1006-骰子大战（一定要好好读题，把题面中的所有限制信息都读到）

题目大意

题目描述
需要构造 nnn 个具有 kkk 个面的骰子，每个面上的数字取值范围为 0∼m0 \sim m0∼m。
两个骰子进行对决时，随机掷出一个面，点数大者获胜。要求构造的骰子组合必须同时满足以下三个条件：

  1. 循环劣势：形成 1→2→3→⋯→n−1→n→11 \to 2 \to 3 \to \dots \to n-1 \to n \to 11→2→3→⋯→n−1→n→1 的胜负关系。即 1 号骰子输给 2 号，2 号输给 3 号，……，nnn 号输给 1 号（“输给”定义为在 k×kk \times kk×k 种可能的掷出组合中，获胜的次数严格少于对方）。

  2. 绝对无平局：每个数字至多只能出现在一个骰子上，保证任意两个骰子对决时不可能掷出相同的点数。

  3. 字典序最小：将 1 号到 nnn 号骰子上的所有数字按顺序拼接成一个长度为 n×kn \times kn×k 的序列，要求该序列在所有合法方案中字典序最小。

输入格式
第一行输入一个正整数 TTT，表示测试数据组数。
接下来每组数据输入三个正整数 nnn、mmm、kkk，分别表示骰子数量、面上的最大允许数字、每个骰子的面数。

输出格式
对于每组数据，输出一行 n×kn \times kn×k 个用空格分隔的整数，依次表示 1 号到 nnn 号骰子各个面上的数字。

样例解释
输入代表需要构造 333 个骰子，每个骰子有 333 个面，数字范围在 0∼40 \sim 40∼4 之间。根据输出样例，三个骰子的构造如下：

  • 1 号骰子：[0, 3, 3]

  • 2 号骰子：[1, 1, 4]

  • 3 号骰子：[2, 2, 2]

胜负关系验证（总对决情况数为 3×3=93 \times 3 = 93×3=9 种）：

  • 1 号 vs 2 号：1 号获胜的情况为 3>1（共 4 次），2 号获胜的情况为 1>0、1>0、4>0、4>3、4>3（共 5 次）。2 号胜率更高，1 号输给 2 号。

  • 2 号 vs 3 号：2 号获胜的情况为 4>2（共 3 次），3 号获胜的情况为 2>1（共 6 次）。3 号胜率更高，2 号输给 3 号。

  • 3 号 vs 1 号：3 号获胜的情况为 2>0（共 3 次），1 号获胜的情况为 3>2（共 6 次）。1 号胜率更高，3 号输给 1 号。

该方案数字各不相同（无平局），构成了 1→2→3→11 \to 2 \to 3 \to 11→2→3→1 的循环劣势，且可以证明拼接后的序列 0 3 3 1 1 4 2 2 2 是所有满足条件的方案中字典序最小的。

这道题是一道非常精妙的构造题，涉及博弈与贪心策略。为了得到字典序最小的方案，我们需要深入分析骰子胜负的数学本质。

题目描述

我们需要构造 nnn 个具有 kkk 个面的骰子，每面的数字范围为 0∼m0 \sim m0∼m。要求形成 1→2→3→⋯→n→11 \to 2 \to 3 \to \dots \to n \to 11→2→3→⋯→n→1 的循环劣势（即 i+1i+1i+1 击败 iii，111 击败 nnn），其中“击败”定义为在 k2k^2k2 次可能对决中获胜次数严格大于一半。不仅要满足该条件，还要使所有骰子的面按顺序拼接后的序列字典序最小。

解题思路

1. 胜负条件的数学转化：
由于不能平局，任意两个骰子对决共有 k2k^2k2 种结果。“BBB 击败 AAA” 等价于 BBB 在对决中获胜的次数 W(B,A)≥⌊k2/2⌋+1W(B, A) \ge \lfloor k^2/2 \rfloor + 1W(B,A)≥⌊k2/2⌋+1。
令目标获胜次数为 w=⌊k2/2⌋+1w = \lfloor k^2/2 \rfloor + 1w=⌊k2/2⌋+1。
2. 字典序最小的贪心框架：
为了让整体拼接成的序列字典序最小，我们需要让 111 号骰子的数字尽可能小，且小数字尽可能多；接着是 222 号骰子，依此类推。
这启发我们设计一种 “两阶段划分（双段构造）” 的模式。将每个骰子 iii 的 kkk 个面分成两部分：

  • 早段（较小的数字）：包含 xix_ixi​ 个面。
  • 晚段（较大的数字）：包含 yi=k−xiy_i = k - x_iyi​=k−xi​ 个面。
    为了最小化序列，我们按照如下顺序分配逐渐递增的数值：
    111 号早段 →2\to 2→2 号早段 ⋯→n\dots \to n⋯→n 号早段 →1\to 1→1 号晚段 →2\to 2→2 号晚段 ⋯→n\dots \to n⋯→n 号晚段。
    同一段内的面赋予完全相同的数字，不同段赋予递增的数字。
    3. 推导核心不等式：
    在上述赋值顺序下，考虑相邻骰子的对决：
  • i+1i+1i+1 击败 iii (1≤i<n1 \le i < n1≤i<n)：
    i+1i+1i+1 的“早段”排在 iii 的“早段”之后，所以能赢 xi+1×xix_{i+1} \times x_ixi+1​×xi​ 次。
    i+1i+1i+1 的“晚段”排在 iii 的两段之后，所以能赢 yi+1×ky_{i+1} \times kyi+1​×k 次。
    总获胜次数：xi+1xi+(k−xi+1)k≥wx_{i+1} x_i + (k - x_{i+1}) k \ge wxi+1​xi​+(k−xi+1​)k≥w
    化简得：xi+1(k−xi)≤k2−wx_{i+1} (k - x_i) \le k^2 - wxi+1​(k−xi​)≤k2−w。令 R=k2−wR = k^2 - wR=k2−w，即 xi+1(k−xi)≤Rx_{i+1} (k - x_i) \le Rxi+1​(k−xi​)≤R。
  • 111 击败 nnn：
    111 的“早段”排在最前面，赢不了 nnn 的任何面。
    111 的“晚段”排在 nnn 的“早段”之后、“晚段”之前，所以能赢 y1×xny_1 \times x_ny1​×xn​ 次。
    总获胜次数：y1xn=(k−x1)xn≥wy_1 x_n = (k - x_1) x_n \ge wy1​xn​=(k−x1​)xn​≥w。
    4. 贪心求解 xix_ixi​：
    要使字典序最小，我们要尽可能让早段容量变大（即尽量最大化 x1,x2,…x_1, x_2, \dotsx1​,x2​,…）：
  • 由 (k−x1)xn≥w(k - x_1) x_n \ge w(k−x1​)xn​≥w，且 xn≤kx_n \le kxn​≤k，可知 (k−x1)k≥w(k - x_1) k \ge w(k−x1​)k≥w，即 x1≤k−⌈w/k⌉x_1 \le k - \lceil w/k \rceilx1​≤k−⌈w/k⌉。我们直接取最大值：x1=k−⌈w/k⌉x_1 = k - \lceil w/k \rceilx1​=k−⌈w/k⌉。
  • 对于后续的 xix_ixi​ (i≥2i \ge 2i≥2)，根据 xi(k−xi−1)≤Rx_i (k - x_{i-1}) \le Rxi​(k−xi−1​)≤R，只要 k−xi−1>0k - x_{i-1} > 0k−xi−1​>0，我们就贪心地取最大合法值：xi=min⁡(k,⌊R/(k−xi−1)⌋)x_i = \min(k, \lfloor R / (k - x_{i-1}) \rfloor)xi​=min(k,⌊R/(k−xi−1​)⌋)。
    （如果 xi−1=kx_{i-1} = kxi−1​=k，意味着 iii 号骰子可以无条件获胜，此时直接设 xi=kx_i = kxi​=k）。
    由于题目保证解存在，mmm 必定足够大，我们只需依据 xix_ixi​ 和 yiy_iyi​ 生成非空的段并依次赋上递增的数字即可。

复杂度分析

  • 时间复杂度：计算所有的段落分布 xix_ixi​ 只需要一重循环遍历 111 到 nnn。为骰子面赋值和打印输出的循环总计处理了 n×kn \times kn×k 次操作。所以处理每组测试数据的时间复杂度为 O(n×k)\mathcal{O}(n \times k)O(n×k)。结合总和 ∑n×k≤2×105\sum n \times k \le 2 \times 10^5∑n×k≤2×105，算法能在极短时间内高速执行完成。
  • 空间复杂度：由于只存储了诸如 x、early_val 和 late_val 这些一维状态数组，占用空间为 O(n)\mathcal{O}(n)O(n)。这远低于题目宽松的 512 MB512 \text{ MB}512 MB 空间限制。

首先，我们为什么会想到双段构造？呃，其实正常的思维链路应该是，我们先想怎么构造，

其实题面里面讲的非常清楚，一个骰子最多只会有两种不同的数字，每个数字仅能出现在至多一个骰子上。

那都说了，最多两个数，那还能不是双段构造吗？

如果分成双段构造，那么不难看出：

早段1<早段2<⋯<早段n\text{早段}_1 < \text{早段}_2 < \dots < \text{早段}_n早段1​<早段2​<⋯<早段n​，$\text{晚段}_1 < \text{晚段}_2 < \dots < \text{晚段}_n $（为了字典序最小，这两个条件是必须的），但是早晚段之间是什么关系呢？

注意啊，为了赢得游戏，早段n<晚段1\text{早段}_n < \text{晚段}_1早段n​<晚段1​，这个也是为了赢得游戏，满足所谓的“循环劣势要求”，所必须的这个构造，一个的最小的，大于一个的最大的，两者之间是绝对不会发生这个穿插的这个情况的，因此就是 早段1<早段2<⋯<早段n<晚段1<晚段2<⋯<晚段n\text{早段}_1 < \text{早段}_2 < \dots < \text{早段}_n < \text{晚段}_1 < \text{晚段}_2 < \dots < \text{晚段}_n早段1​<早段2​<⋯<早段n​<晚段1​<晚段2​<⋯<晚段n​。
