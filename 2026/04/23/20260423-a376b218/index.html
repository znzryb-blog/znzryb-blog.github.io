题目大意

题面

Alice 手上有两个长度为 NNN 的二进制字符串 AAA 和 BBB（仅由字符 0 和 1 组成）。

Alice 玩如下游戏。初始时她的分数为 000。每次迭代：

  • 她把两个字符串的首字符乘积加到分数上，即分数增加 A1⋅B1A_1 \cdot B_1A1​⋅B1​。

  • 如果 A1=B1A_1 = B_1A1​=B1​，就把 AAA 替换为 left_shift(A)\mathrm{left\_shift}(A)left_shift(A)。

  • 否则把 BBB 替换为 left_shift(B)\mathrm{left\_shift}(B)left_shift(B)。

对于二进制字符串 S=S1S2…SLS = S_1 S_2 \ldots S_LS=S1​S2​…SL​，left_shift(S)\mathrm{left\_shift}(S)left_shift(S) 定义为删去首字符并追加到末尾：

left_shift(S)=S2S3…SLS1\mathrm{left\_shift}(S) = S_2 S_3 \ldots S_L S_1 left_shift(S)=S2​S3​…SL​S1​

现在有 QQQ 次独立询问。第 iii 次询问给出整数 KiK_iKi​，需要求出恰好执行 KiK_iKi​ 次迭代后 Alice 的分数。各次询问彼此独立，每次都视为从初始字符串 AAA、BBB 重新开始。

输入格式

  • 第一行一个整数 TTT，表示测试用例组数。

  • 每组测试用例共四行：

    • 第一行两个整数 NNN、QQQ，分别表示字符串长度与询问次数。
    • 第二行是二进制字符串 AAA。
    • 第三行是二进制字符串 BBB。
    • 第四行 QQQ 个空格分隔的整数 K1,K2,…,KQK_1, K_2, \ldots, K_QK1​,K2​,…,KQ​。

输出格式

对每组测试用例输出一行，包含 QQQ 个空格分隔的整数，第 iii 个整数是执行 KiK_iKi​ 次迭代后 Alice 的分数。

数据范围

  • 1≤T≤1041 \leq T \leq 10^41≤T≤104

  • 1≤N,Q≤1061 \leq N, Q \leq 10^61≤N,Q≤106

  • 1≤Ki≤10121 \leq K_i \leq 10^{12}1≤Ki​≤1012

  • 所有测试用例的 NNN 之和、QQQ 之和各自不超过 10610^6106

  • 时间限制 333 秒；内存限制 1.51.51.5 GB

思路讲解

一句话

KKK 最大到 101210^{12}1012，逐步模拟必炸；这题的结构是「run-length 周期 + 首块 + 整圈 + 零头」三段式，外面再套个二分。

状态机：matched / mismatched

得分 A1⋅B1A_1 \cdot B_1A1​⋅B1​ 只有在 A1=B1=1A_1 = B_1 = 1A1​=B1​=1 时是 1，其他都是 0。而且「谁前进」完全绑定在 A1=?B1A_1 \stackrel{?}{=} B_1A1​=?B1​ 上，所以过程天然分两态：

  • matched（A1=B1A_1 = B_1A1​=B1​）：左移 AAA，得分 =A1= A_1=A1​

  • mismatched（A1≠B1A_1 \ne B_1A1​=B1​）：左移 BBB，得分 =0= 0=0

matched 段持续到 AAA 跨过一个 AAA-run（A1A_1A1​ 翻转），mismatched 段持续到 BBB 跨过一个 BBB-run（B1B_1B1​ 翻转），两态在 run 边界交替。

Phase：一个 matched + 一个 mismatched

把相邻的 matched + mismatched 合成一个 phase。每个 phase 吃掉 AAA 的 1 个 run + BBB 的 1 个 run，步数 =len(A-run)+len(B-run)= \mathrm{len}(A\text{-run}) + \mathrm{len}(B\text{-run})=len(A-run)+len(B-run)。

关键不变量：

每过一个 phase，A1,B1A_1, B_1A1​,B1​ 各翻 1 次，相对关系还原——所以所有 phase 的内部子结构都和第 1 个一样，只需判一次 cA=?cBc_A \stackrel{?}{=} c_BcA​=?cB​（cA,cBc_A, c_BcA​,cB​ 就是 A1,B1A_1, B_1A1​,B1​ 的初始值）。后面零头分类直接吃这条不变量。

跳过首块 + 周期性

AAA 从首块内部起步，所以首块是残缺的（长度 L0AL_0^AL0A​，与 A1A_1A1​ 同字符的极长前缀）。跳过首块后从 L0A mod NL_0^A \bmod NL0A​modN 开始把 AAA 当循环串扫一圈，得到完整 run 长度序列 r0A,r1A,…,rkA−1Ar^A_0, r^A_1, \ldots, r^A_{k_A-1}r0A​,r1A​,…,rkA​−1A​，段长总和 =N= N=N，之后按段数 kAk_AkA​ 为周期重复。BBB 侧同构。

注意 段数 kAk_AkA​ 和段数 kBk_BkB​ 一般不相等，两套预处理彼此独立——同一个 PPP（已过 phase 数）在 AAA 侧和 BBB 侧走的圈数、零头段号可以完全不一样。

三段式 O(1)O(1)O(1) 函数

走完 PPP 个 phase === phase 1 吃首块 +++ 剩下 P−1P - 1P−1 个 phase 按周期跑完。两侧各自拆 P−1=q⋅k+rP - 1 = q \cdot k + rP−1=q⋅k+r：

fA(P)  =  L0A+qA⋅N+pref_a[rA]f_A(P) \;=\; L_0^A + q_A \cdot N + \mathrm{pref\_a}[r_A] fA​(P)=L0A​+qA​⋅N+pref_a[rA​]

fB(P)  =  L0B+qB⋅N+pref_b[rB]f_B(P) \;=\; L_0^B + q_B \cdot N + \mathrm{pref\_b}[r_B] fB​(P)=L0B​+qB​⋅N+pref_b[rB​]

gA(P)  =  L0A⋅cA+qA⋅WA+pref_score_A[rA]g_A(P) \;=\; L_0^A \cdot c_A + q_A \cdot W_A + \mathrm{pref\_score\_A}[r_A] gA​(P)=L0A​⋅cA​+qA​⋅WA​+pref_score_A[rA​]

WA=∑iriA⋅rateiW_A = \sum_i r^A_i \cdot \mathrm{rate}_iWA​=∑i​riA​⋅ratei​ 是一整圈的得分（ratei\mathrm{rate}_iratei​ 就是第 iii 段 A1A_1A1​ 的值：偶 iii 是 1−cA1 - c_A1−cA​、奇 iii 是 cAc_AcA​）。P=0P = 0P=0 要直接 return 0，别套公式——不然首块项会白加，而且 P−1=−1P - 1 = -1P−1=−1 做下取整和取模在 C++ 里行为还依赖符号约定，徒增隐患。

二分 + 零头（这个二分应该是单纯我们不大能推出来这个最大能走）

fA(P)+fB(P)f_A(P) + f_B(P)fA​(P)+fB​(P) 关于 PPP 单调递增，直接二分找最大 P∗P^*P∗ 使 fA(P∗)+fB(P∗)≤Kf_A(P^*) + f_B(P^*) \le KfA​(P∗)+fB​(P∗)≤K。单查询 O(log⁡K)O(\log K)O(logK)。

零头 R=K−(fA(P∗)+fB(P∗))R = K - (f_A(P^*) + f_B(P^*))R=K−(fA​(P∗)+fB​(P∗)) 落在第 P∗+1P^* + 1P∗+1 个 phase 里——用那个不变量，查一次 cA=?cBc_A \stackrel{?}{=} c_BcA​=?cB​ 定案：

cAc_AcA​ vs cBc_BcB​ phase 内部结构 RRR 步贡献
cA=cBc_A = c_BcA​=cB​ 先 matched 后 mismatched min⁡(R, lenA)⋅rateA\min(R,\ \mathrm{len}_A) \cdot \mathrm{rate}_Amin(R, lenA​)⋅rateA​
cA≠cBc_A \ne c_BcA​=cB​ 先 mismatched 后 matched max⁡(R−lenB, 0)⋅rateA\max(R - \mathrm{len}_B,\ 0) \cdot \mathrm{rate}_Amax(R−lenB​, 0)⋅rateA​

口诀：cA=cBc_A = c_BcA​=cB​ 时 AAA 在前，RRR 先花在 matched 段；cA≠cBc_A \ne c_BcA​=cB​ 时 BBB 在前，要先熬过 lenB\mathrm{len}_BlenB​ 步 mismatched 才落到 matched。代码里 A[0] == B[0] 就是在判这个。

uniform 特判

AAA 或 BBB 全同字符时翻转结构退化（kA=0k_A = 0kA​=0 之类），奇偶 rate 公式直接出事。把「都 uniform / 只 AAA uniform / 只 BBB uniform」三类单独闭式处理绕开。通解写完一定要回头扫一遍退化分支，不然很容易暴毙。

复杂度

预处理 O(N)O(N)O(N)，单次询问 O(log⁡K)O(\log K)O(logK)，总 O(N+Qlog⁡K)≈106+106⋅40≈4×107O(N + Q \log K) \approx 10^6 + 10^6 \cdot 40 \approx 4 \times 10^7O(N+QlogK)≈106+106⋅40≈4×107，稳过 3 s。
