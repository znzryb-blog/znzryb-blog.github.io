CF 1095D - Reserved Reversals

Posted on 2026-04-29 Edited on 2026-05-02

🤖 Claude 苏格拉底式带教对话录原稿。每一步都是 mentor 提一个聚焦问题 → me 答 → 反复推进至完整算法。蓝色 callout 是 Claude，橙色 callout 是 zzy，原稿里 mentor 的反例 / 矫枉过正 / 用户吐槽全部逐字保留。

原题：CF 1095D Reserved Reversals（Codeforces Round 1095 Div.2）

第 1 步：理解题目

🤖 Note: Claude (Mentor)
先确认操作语义。题面给的合法操作是：选 [l,r][l, r][l,r] 满足 min⁡(al…ar)+max⁡(al…ar)\min(a_l… a_r) + \max(a_l… a_r)min(al​…ar​)+max(al​…ar​) 是奇数，然后把这一段反转。
样例 5 是 a=[4,2,4,2,4]a = [4, 2, 4, 2, 4]a=[4,2,4,2,4] ，题面提示这个 case 答案 NO 的理由是"找不到任何合法的 (l,r)(l, r)(l,r) "。
问题：对这个具体数组，为什么任何 (l,r)(l, r)(l,r) 都不合法？用一句话说出 “min+max 奇数” 这条约束实际上等价于 segment 满足什么条件。

🧑 Note: zzy (Me)
对于一个具体数组来说，如果找不到任何一个这个操作序列，那么就说明数组全部为奇数，或者全部为偶数是吧

🤖 Note: Claude (Mentor)
对。再精确一点： min⁡+max⁡\min + \maxmin+max 奇 ⇔\Leftrightarrow⇔ min⁡\minmin 与 max⁡\maxmax 一奇一偶 ⇔\Leftrightarrow⇔ 这个 segment 至少含一个奇数且至少含一个偶数（因为 min/max 都来自 segment 内部，segment 全奇 ⇒\Rightarrow⇒ min/max 都奇 ⇒\Rightarrow⇒ 和偶；全偶同理）。
所以"找不到任何合法 (l,r)(l, r)(l,r) “等价于"整个数组单一奇偶性”。样例 5 全偶就是这样。
记一个名字方便后面提：把含 “至少一奇一偶” 的 segment 叫做混色 segment（mixed-parity segment）。只有混色 segment 才能反转。
接下来对一下数据范围： 1≤n≤2⋅1051 \le n \le 2 \cdot 10^51≤n≤2⋅105 ， ∑n≤2⋅105\sum n \le 2 \cdot 10^5∑n≤2⋅105 ，时限 222 s， 1≤ai≤n1 \le a_i \le n1≤ai​≤n 。
问题：你想往什么时间复杂度量级走？

🧑 Note: zzy (Me)
这个显然是 O(nlog⁡n)O(n\log n)O(nlogn) 的这个时间复杂度

🤖 Note: Claude (Mentor)
合理。 n≤2⋅105n \le 2 \cdot 10^5n≤2⋅105 ， O(n2)=4⋅1010O(n^2) = 4 \cdot 10^{10}O(n2)=4⋅1010 直接寄，目标 O(nlog⁡n)O(n \log n)O(nlogn) 或更优（很可能 O(n)O(n)O(n) 就能搞定）。

第 2 步：找判定条件（不变量）

🤖 Note: Claude (Mentor)
正常套路是"暴力 → 优化"，但这题暴力 BFS 不现实——状态空间是 aaa 的所有排列， nnn 一大就爆。所以得直接找结构性判定条件：什么样的 aaa 是 YES，什么样的 NO。
样例 5 [4,2,4,2,4][4,2,4,2,4][4,2,4,2,4] 全偶 ⇒\Rightarrow⇒ 没合法操作 ⇒\Rightarrow⇒ 没排好序就 NO，简单。但样例 6 a=[3,3,1,5,5,2]a = [3, 3, 1, 5, 5, 2]a=[3,3,1,5,5,2] 也是 NO——这个数组有 5 个奇 +++ 1 个偶（位置 6 那个 222 ），随便找含 222 的段就能凑出混色段，明明可以做操作，为啥是 NO？
问题：手动模拟一下 [3,3,1,5,5,2][3, 3, 1, 5, 5, 2][3,3,1,5,5,2] ，挑几个你觉得有用的合法 (l,r)(l, r)(l,r) 反转看看，看能不能感知出"这个 222 跑来跑去能不能把数组排好序"——以及为啥不能。

🧑 Note: zzy (Me)
肯定是不能的，因为 1 没有办法跑到最前面，1 要跑到最前面，肯定要有一个比较大的偶数，但是最大的偶数就是 2，在 3 的阻挠下，1 是没有办法跑到前面去的

🤖 Note: Claude (Mentor)
对，"在 3 的阻挠下，1 跑不到前面"就是核心不变量的雏形。把它推广一步：
问题：考虑数组里两个同奇偶的值 a<ba < ba<b 。在什么条件下， aaa 和 bbb 的相对顺序在所有合法操作下都不可能改变？（即：如果原数组里 bbb 在 aaa 前面，那永远没法变成 aaa 在 bbb 前面。）

🧑 Note: zzy (Me)
这个有点困难，我想的就是，如果任意包含 a,ba, ba,b 的子数组中，满足可交换条件即可，具体的话，我就不细讲，总的来说就是最大值和最小值的这个奇偶性要不同

🤖 Note: Claude (Mentor)
对，框架对了。我来补完最后一步形式化：
要 swap a,ba, ba,b ，必须有合法反转 [l,r][l, r][l,r] 同时包含两人位置。这种反转的 min⁡≤a\min \le amin≤a 、 max⁡≥b\max \ge bmax≥b 。合法 ⇔min⁡+max⁡\Leftrightarrow \min + \max⇔min+max 奇 ⇔\Leftrightarrow⇔ 一奇一偶。但 a,ba, ba,b 同奇偶，所以 min⁡\minmin 和 max⁡\maxmax 至少有一个不能是 aaa 也不能是 bbb ，必须是某个"更极端"的异奇偶值 ccc ，且 c<ac < ac<a （充当 min⁡\minmin ）或 c>bc > bc>b （充当 max⁡\maxmax ）。
所以 locked 条件（同奇偶 a<ba < ba<b 永远换不了相对顺序）：

数组里没有任何异奇偶值 c 满足 c<a 或 c>b\boxed{\text{数组里没有任何异奇偶值 } c \text{ 满足 } c < a \text{ 或 } c > b} 数组里没有任何异奇偶值 c 满足 c<a 或 c>b​

等价地：所有异奇偶值都落在闭区间 [a,b][a, b][a,b] 内。
设 EEE 表数组里偶数集合、 OOO 表奇数集合（异奇偶视情况取另一个）。对同为奇数的 a<ba < ba<b ：locked ⇔min⁡(E)≥a\Leftrightarrow \min(E) \ge a⇔min(E)≥a 且 max⁡(E)≤b\max(E) \le bmax(E)≤b （约定 E=∅E = \emptysetE=∅ 时 min⁡=+∞\min = +\inftymin=+∞ 、 max⁡=−∞\max = -\inftymax=−∞ ，那么 vacuously locked）。同奇偶为偶时对称。
样例 6 验证： O={1,3,3,5,5},E={2}O = \{1, 3, 3, 5, 5\}, E = \{2\}O={1,3,3,5,5},E={2} ， min⁡(E)=max⁡(E)=2\min(E) = \max(E) = 2min(E)=max(E)=2 。

  • (1,3)(1, 3)(1,3) ： 2≥12 \ge 12≥1 ✓， 2≤32 \le 32≤3 ✓ ⇒\Rightarrow⇒ locked。原数组 111 在两个 333 之后，sorted 要 111 在前 ⇒\Rightarrow⇒ 违反 ⇒\Rightarrow⇒ NO ✓
  • (1,5)(1, 5)(1,5) ：locked，同样违反。
  • (3,5)(3, 5)(3,5) ： 2≥32 \ge 32≥3 ✗ ⇒\Rightarrow⇒ 不 locked，可换。
    问题：这个 locked 条件给了一个必要条件（违反就 NO）。它是不是也充分？换句话说：如果对所有 locked 同奇偶对 (a,b)(a, b)(a,b) 、原数组里 aaa 都已经在 bbb 前，是否一定 YES？想想看，怎么说服自己。

🧑 Note: zzy (Me)
这个还能不充分？我认为，首先要确认这个问题，问题就是， b>ab > ab>a ，而且这个先 bbb 后 aaa ，永远也解决不了，只要确认了有这样的对存在，那么肯定是这个错的。
不过，我现在理解你的意思了，上面这段话，就说明了其是一个必要条件，但是没法说明这个是一个充分条件。
呃，不过也只能猜一手了，我觉得这个条件挺对的

🤖 Note: Claude (Mentor)
对，这个条件确实也充分（建立在数据范围内的合法操作总能凑出来）。直觉理由：

  • 异奇偶相邻 swap 永远合法（长度 2 的混色段反转 = 直接交换），所以 "奇偶交错"的局部错位都可以 bubble sort 修。
  • 任何未 locked 同奇偶对 (a,b)(a, b)(a,b) 都存在某个"更极端"的异奇偶值 ccc ，可以选一个含 a,b,ca, b, ca,b,c 的反转、把 ccc 抬成 min⁡\minmin 或 max⁡\maxmax 就把 a,ba, ba,b 交换过来。
  • Locked 对原本就已正确（因为题设保证），不需要动。
    严格证明要构造性 sorting 流程，这里就跳过——竞赛里这种"sortability" 题判定通常都是: 必要条件 = 充分条件，信一手即可，对拍验证就稳了。

第 3 步：把判定条件落到 O(n)O(n)O(n) 算法

🤖 Note: Claude (Mentor)
Locked 条件复述：同奇偶对 (a,b), a<b(a, b),\ a < b(a,b), a<b 锁死 ⇔\Leftrightarrow⇔ 异奇偶集合 EEE 满足 min⁡(E)≥a\min(E) \ge amin(E)≥a 且 max⁡(E)≤b\max(E) \le bmax(E)≤b 。“判定 YES/NO” = 对每个 locked 对，检查原数组里 aaa 所有位置 <<< bbb 所有位置。
朴素地枚举所有同奇偶对就是 O(n2)O(n^2)O(n2) 。但 locked 对其实有非常规整的结构——它们不是"任意"的同奇偶对。
问题：固定奇偶 = 奇（讨论奇与奇之间锁死的 pair）。设异奇偶（偶数）集合的 min⁡(E)=M−\min(E) = M_-min(E)=M−​ 、 max⁡(E)=M+\max(E) = M_+max(E)=M+​ 。把奇值按 ≤M−\le M_-≤M−​ 、 ∈(M−,M+)\in (M_-, M_+)∈(M−​,M+​) 、 ≥M+\ge M_+≥M+​ 分三段——记为 LLL （low）/ MMM （middle）/ HHH （high）。
哪些 locked 奇-奇对都是属于哪两段之间的？想清楚这一步，你应该能直接给出一个 O(n)O(n)O(n) 判定算法。

对每个奇偶 ppp ：设异奇偶集合 EpE_pEp​ ，记 M−=min⁡(Ep), M+=max⁡(Ep)M_- = \min(E_p),\ M_+ = \max(E_p)M−​=min(Ep​), M+​=max(Ep​) 。

  • Ep=∅E_p = \emptysetEp​=∅ ：奇偶 ppp 子序列必须已非递减，否则 NO。
  • 否则记 lastLp=max⁡{i:ai 奇偶=p, ai≤M−}\mathrm{lastL}_p = \max\{i : a_i \text{ 奇偶}=p,\ a_i \le M_-\}lastLp​=max{i:ai​ 奇偶=p, ai​≤M−​} 、 firstHp=min⁡{i:ai 奇偶=p, ai≥M+}\mathrm{firstH}_p = \min\{i : a_i \text{ 奇偶}=p,\ a_i \ge M_+\}firstHp​=min{i:ai​ 奇偶=p, ai​≥M+​} 。两者皆存在且 lastLp≥firstHp⇒\mathrm{lastL}_p \ge \mathrm{firstH}_p \RightarrowlastLp​≥firstHp​⇒ NO。

心路历程（WA → AC）

第一版代码两个算法层 bug：

  1. findGloC 两个 arg 接的端点反了——findL 应该喂小端、findR 应该喂大端，但喂反成 findL(big) || findR(small)。这个 expression 在异奇偶非空时恒为 true，等价于"opp 非空就 unlocked"——退化得只有"单一奇偶且非递减"才 NO，3 / 6 都漏判。

  2. 修了 arg 顺序后，findNearNum 取最近的更小同奇偶（即"最大的更小"）作为 inversion 的小端。但 locked 配对的小端要求 a≤M−a \le M_-a≤M−​ ，"最近的更小"可能落在中间段 MMM 里，配 A[i]A[i]A[i] 不 locked，但 stMat 里还藏着真正在 LLL 段的更小值——它配 A[i]A[i]A[i] 才 locked。改成 *stMat[par].begin()（stMat 里最小的同奇偶）后再判，过样例 1 但 sample 2 stress test 第一组 n=1 [1] 又挂了——findSmallNum 没排除 “smallest >= num” 的退化情形（n=1 时 stMat 只有自己，最小值 = num，被当成 inversion 触发了不该跑的 locked 测试）。最终修法：*it < num 才返，否则 INF。
