CF 1094C Median Partition

Posted on 2026-04-26

题目大意

给定一个长度为奇数 nnn 的正整数数组 aaa，把它划分成若干奇长度子数组，要求每段中位数都相同。求最大段数 ppp。

形式化：求严格递增序列 k1=1,k2,…,kp+1=n+1k_1=1, k_2, \ldots, k_{p+1}=n+1k1​=1,k2​,…,kp+1​=n+1，对每个 iii，子数组 [aki,…,aki+1−1][a_{k_i}, \ldots, a_{k_{i+1}-1}][aki​​,…,aki+1​−1​] 中位数都相同；且 ki,ki+1k_i, k_{i+1}ki​,ki+1​ 奇偶性不同（这条等价于段长为奇）。最大化 ppp。

数据范围

  • T≤1000T \le 1000T≤1000

  • 1≤n<50001 \le n < 50001≤n<5000，nnn 为奇数

  • 1≤ai≤1091 \le a_i \le 10^91≤ai​≤109

  • ∑n2≤50002\sum n^2 \le 5000^2∑n2≤50002

  • 时限 2s，内存 256 MB

都吃下。两条都对——上面那段是按我自己的语义重写了你的 dp，确实越界。重新答你的通用问题：
只要 dp[i] 按「末元素下标」索引，转移到 dp[j-1] 时 j=0 必然撞 dp[-1]（空前缀的 base）。这个 base 不是某题特例，是所有按位置索引的 DP 的共性。通用治法只有两种范式：
(A) Sentinel 包装，保留你的语义
不改 dp 的语义，包一层访问器：

或直接内联三元：

你说"三元本质上还是 if-else"——对，是。但 base 的判断逻辑上必须存在：要么写出来要么藏到访问器里。Sentinel 的好处是包装一次、之后转移式干净没补丁。
(B) 索引整体右移 1，吃掉分支
机械地把 dp[i] 全部重命名成 dp'[i+1]：分配大小 N+1，dp'[0] 就是空前缀的 base，不需要负下标。这不是改你的语义，是纯下标偏移：

很多人看到这写法叫"按前缀长度索引"——其实只是机械的 +1 偏移，不是换语义。它的好处是热循环没分支。
两种等价，挑顺手的。(A) 贴近你的思维模型，(B) 编译出的代码可能稍快一丁点（其实差不多）。

心路历程

从 mentor.tex 对话录里提炼出的 3 次主要卡壳：

  • **第一次摔：**理解错题意，以为子数组可以重叠（其实是滑动窗口的概念）。题面里 k 序列严格递增、首尾相接才是合法划分——每个元素恰好属于一段。

  • **第二次摔：**写区间中位数判定时想用「区间内 m 的个数 > ⌈len/2⌉」当条件，被反例 [1,2,3],m=2[1,2,3], m=2[1,2,3],m=2 打回——这是「严格多数」、比「中位数 = m」强。换成 ±1 编码 + 两套前缀和才对。

  • **第三次摔：**dp 默认 0 初始化，dp[r+1]=max⁡(…,1+dp[l])\text{dp}[r+1] = \max(\ldots, 1 + \text{dp}[l])dp[r+1]=max(…,1+dp[l]) 把不可达的 dp[l] = 0 当成合法 base，段数被高估。改成 dp(N+1, -INF); dp[0] = 0 后 AC。

关键教训写进了板子 §debug 章节：DP 不可达状态必须用 -INF 显式标识——base 之外一律 -INF（求 max）/ INF（求 min），0 只能用于「计数 / 无贡献」语义。
