题目大意

题目大意

Alice 和 Bob 在玩一个游戏。给定两个数组：包含 nnn 个元素的数组 aaa 和包含 mmm 个元素的数组 bbb。

两人轮流进行操作，Alice 先手。在每个回合中，玩家需要从数组 aaa 中选择一个数字 xxx，并从数组 bbb 中选择一个数字 yyy。

  • Alice 的规则：选择的 yyy 必须能被 xxx 整除。

  • Bob 的规则：选择的 yyy 必须不能被 xxx 整除。

选定 xxx 和 yyy 后，yyy 会从数组 bbb 中被移除（如果数组中存在多个相同的 yyy，只移除其中一个），而 xxx 仍然保留在数组 aaa 中。无法进行合法操作（即无法选出满足自身规则的 xxx 和 yyy）的玩家将输掉游戏。

假设双方都采取最优策略，请判断谁会赢得游戏。

输入格式

第一行包含一个整数 ttt（1≤t≤1041 \le t \le 10^41≤t≤104），表示测试用例的数量。

对于每个测试用例：
第一行包含两个整数 nnn 和 mmm（1≤n,m≤1061 \le n, m \le 10^61≤n,m≤106）。
第二行包含 nnn 个整数 aia_iai​（1≤ai≤n+m1 \le a_i \le n+m1≤ai​≤n+m），表示数组 aaa 的元素。
第三行包含 mmm 个整数 bib_ibi​（1≤bi≤n+m1 \le b_i \le n+m1≤bi​≤n+m），表示数组 bbb 的元素。

保证所有测试用例中 nnn 的总和以及 mmm 的总和均不超过 10610^6106。

输出格式

对于每个测试用例，输出获胜者的名字：如果 Alice 赢，输出 Alice；如果 Bob 赢，输出 Bob。

样例解释

对于第一个测试用例，Alice 的获胜过程如下：

  1. Alice 的回合：选择 x=3,y=6x=3, y=6x=3,y=6（666 能被 333 整除）。操作后 666 从数组 bbb 中移除，此时 b=[7,12]b=[7, 12]b=[7,12]。

  2. Bob 的回合：选择 x=3,y=7x=3, y=7x=3,y=7（777 不能被 333 整除）。Bob 在这个回合只能选择 y=7y=7y=7，因为对于 y=12y=12y=12，数组 aaa 中找不到任何不能整除它的 xxx。操作后 777 从数组 bbb 中移除，此时 b=[12]b=[12]b=[12]。

  3. Alice 的回合：选择 x=4,y=12x=4, y=12x=4,y=12（121212 能被 444 整除）。操作后 121212 从数组 bbb 中移除，此时数组 bbb 变为空。

  4. Bob 的回合：数组 bbb 已经为空，Bob 无法进行任何操作，因此 Bob 输掉游戏，Alice 获胜。

思路讲解

OK，我们说回来。这个D为什么交了5发还是没过呢？主要是因为就是这个D，它的数据量比较大。使用Victor victor或者victor的这种调和级数写法呢，它这个时间会超。可以使用vis 数组标记所有 a 的倍数。这样子可以避免卡常。

然后为什么标题里面提了不要用 while 循环？是因为涉及累加的逻辑（在 D 的对拍暴力程序当中），直接 continue 以后，while 循环的这个累加就崩掉了。
