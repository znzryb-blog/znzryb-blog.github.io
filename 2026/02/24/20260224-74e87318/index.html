Codeforces Round 1082 (Div. 2)-CF-1082-E. Rigged Bracket Sequence（括号串）（对括号串一个比较有用的工具就是这个把括号看成 1，-1，合法的括号串就是其前缀和时刻都不为负数）

题目大意

题目描述

给定一个长度为 nnn 的合法括号序列 SSS。

我们定义对 SSS 的一个子序列进行向右循环移位操作：选中子序列所在的位置 i1,i2,…,iki_1, i_2, \dots, i_ki1​,i2​,…,ik​，将这些位置上的字符同时更新为：

  • Si1←SikS_{i_1} \leftarrow S_{i_k}Si1​​←Sik​​

  • Si2←Si1S_{i_2} \leftarrow S_{i_1}Si2​​←Si1​​

  • Si3←Si2S_{i_3} \leftarrow S_{i_2}Si3​​←Si2​​

  • …\dots…

  • Sik←Sik−1S_{i_k} \leftarrow S_{i_{k-1}}Sik​​←Sik−1​​

也就是说，选中的这些位置上的字符整体向右移动一位，最后一个字符移动到第一个被选中的位置。

例如，对于括号序列 ()(()())：

  • 若选中第 222 和第 444 个位置的字符（分别是 ) 和 (），移位后第 222 个位置变为 (，第 444 个位置变为 )，序列变成 ((())())。

  • 若选中第 222、333、555 个位置的字符（分别是 )、(、)），移位后第 222 个位置变为 )，第 333 个位置变为 )，第 555 个位置变为 (，序列变成 ())((())。

请计算有多少个非空子序列，使得在对其进行向右循环移位后，原序列 SSS 仍然是一个合法的括号序列。由于答案可能很大，请输出其对 998244353998244353998244353 取模的结果。

注：两个子序列如果选中的位置集合不同，则被认为是不同的子序列。

输入格式

第一行包含一个整数 ttt（1≤t≤1041 \le t \le 10^41≤t≤104），表示测试用例的数量。
每个测试用例包含两行：

  • 第一行包含一个偶数 nnn（2≤n≤3000002 \le n \le 3000002≤n≤300000），表示括号序列的长度。

  • 第二行包含一个长度为 nnn 的合法括号序列 SSS。
    保证所有测试用例中 nnn 的总和不超过 300000300000300000。

输出格式

对于每个测试用例，在一行中输出一个整数，表示满足条件的非空子序列数量对 998244353998244353998244353 取模的结果。

样例解释

对于第二个测试用例 ()()，以下 888 个非空子序列在向右移位后能使原序列保持合法：

  • 仅选中位置 111：移位后序列为 ()()

  • 仅选中位置 333：移位后序列为 ()()

  • 选中位置 111 和 333：移位后序列为 ()()

  • 选中位置 222 和 333：移位后序列为 (())

  • 选中位置 222 和 444：移位后序列为 ()()

  • 选中位置 111、222、333：移位后序列为 (())

这个就是所谓的right shift操作，那么其实就是向右循环移动一位。题目给你一个正规合法的括号串，问你在所有子序列当中，哪些子序列在进行了right shift操作以后还是合法的括号串（注意，子序列进行了right shift操作后，会反作用于原来的正规合法括号串，那么我们就是要看原来的正规合法括号串，在进行了这个操作以后是否还合法）。

对括号串一个比较有用的工具就是这个把括号看成 1，-1，合法的括号串就是其前缀和时刻都不为负数。（当然末尾必须是 0）
