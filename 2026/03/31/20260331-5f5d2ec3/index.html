题目大意

题目描述总结

给定一个长度为 nnn 的数列 a1,a2,…,ana_1, a_2, \dots, a_na1​,a2​,…,an​ 和一个定值 xxx。两名玩家（铃仙和因幡帝）交替从数列中取数，取出的数会立刻从原数列中删除。规则如下：

  1. 铃仙先手，每次可以从当前数列中任选一个数。

  2. 因幡帝后手，每次必须选择当前数列头部的前 xxx 个数（即剩下数列的第 111 到第 xxx 个数）。若当前数列剩余元素不足 xxx 个，则因幡帝会将剩余的所有数全部取走。

  3. 两人交替选数，直到数列为空。

铃仙的目标是最大化自己所取出的所有数字之和。
假设铃仙始终采取最优策略，请对于所有的 x=1,2,3,…,n−1x = 1, 2, 3, \dots, n-1x=1,2,3,…,n−1，分别独立求出铃仙所能获得的数字和的最大值。

输入格式
第一行包含一个整数 ttt（1≤t≤10001 \le t \le 10001≤t≤1000），表示测试用例数量。
每个测试用例包含两行：
第一行是一个整数 nnn（2≤n≤2×1052 \le n \le 2 \times 10^52≤n≤2×105），表示数列长度。
第二行是 nnn 个正整数 aia_iai​（1≤ai≤1091 \le a_i \le 10^91≤ai​≤109），表示给定的数列。
保证所有测试用例中 nnn 的总和不超过 5×1055 \times 10^55×105。

输出格式
对于每个测试用例输出一行，包含 n−1n-1n−1 个正整数。第 iii 个整数表示当 x=ix=ix=i 时，铃仙能获得的最大数字和。

在样例 222 中，初始数列为 1,1,4,5,1,41, 1, 4, 5, 1, 41,1,4,5,1,4。

当 x=1x=1x=1 时，一种使得铃仙获得最大和的方案为：

  • 铃仙先选取 a4=5a_4=5a4​=5，删除该数，此时数列变为 1,1,4,1,41, 1, 4, 1, 41,1,4,1,4；

  • 因幡帝必须选取头部的 111 个元素 a1=1a_1=1a1​=1，此时数列变为 1,4,1,41, 4, 1, 41,4,1,4；

  • 铃仙再选取 a2=4a_2=4a2​=4，此时数列变为 1,1,41, 1, 41,1,4；

  • 因幡帝必须选取头部的 111 个元素 a1=1a_1=1a1​=1，此时数列变为 1,41, 41,4；

  • 铃仙再选取 a2=4a_2=4a2​=4，此时数列变为 111；

  • 因幡帝必须选取头部的 111 个元素 a1=1a_1=1a1​=1，此时数列为空，停止。
    铃仙所选之数的和为 5+4+4=135+4+4=135+4+4=13。可以证明，该方案铃仙所选之数的和最大。

当 x=3x=3x=3 时，一种使得铃仙获得最大和的方案为：

  • 铃仙先选取 a4=5a_4=5a4​=5，删除该数，此时数列变为 1,1,4,1,41, 1, 4, 1, 41,1,4,1,4；

  • 铃仙再选取 a2=4a_2=4a2​=4，此时数列变为 111；

  • 因幡帝必须选取头部的 333 个元素，但当前数列大小小于 333，全部取完，此时数列为空，停止。
    铃仙所选之数的和为 5+4=95+4=95+4=9。可以证明，该方案铃仙所选之数的和最大。
