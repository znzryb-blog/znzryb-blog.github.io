题目大意

题目描述

有 nnn 篇博客，第 iii 篇博客按顺序提到了 lil_ili​ 个用户，用数组 ai=[ai,1,ai,2,…,ai,li]a_i=[a_{i,1},a_{i,2},\dots,a_{i,l_i}]ai​=[ai,1​,ai,2​,…,ai,li​​] 表示。

你需要将这 nnn 篇博客全部发布。维护一个序列 QQQ（初始为空）来记录最近提到的用户列表。你需要选择一个所有 nnn 篇博客的发布顺序进行发布，每次发布一篇博客 iii 时，会对每一个 1≤j≤li1 \le j \le l_i1≤j≤li​ 依次执行以下操作：

  • 如果 ai,ja_{i,j}ai,j​ 已经存在于 QQQ 中，则将 ai,ja_{i,j}ai,j​ 移动到 QQQ 的开头。

  • 否则，将 ai,ja_{i,j}ai,j​ 插入到 QQQ 的开头。

求在发布所有 nnn 篇博客后，所能得到的字典序最小的序列 QQQ。

输入格式

第一行包含一个整数 ttt（1≤t≤10001 \le t \le 10001≤t≤1000），表示测试用例的数量。
每个测试用例第一行包含一个整数 nnn（1≤n≤30001 \le n \le 30001≤n≤3000），表示博客数量。
接下来 nnn 行，每行首先包含一个整数 lil_ili​（1≤li≤30001 \le l_i \le 30001≤li​≤3000），表示第 iii 篇博客提到的用户数量，随后跟着 lil_ili​ 个整数 ai,1,ai,2,…,ai,lia_{i,1},a_{i,2},\dots,a_{i,l_i}ai,1​,ai,2​,…,ai,li​​（1≤ai,j≤1061 \le a_{i,j} \le 10^61≤ai,j​≤106），表示提到的用户列表。
保证所有测试用例中 nnn 的总和不超过 300030003000，∑li\sum l_i∑li​ 的总和不超过 300030003000。

输出格式

对于每个测试用例，输出一行包含若干个整数，表示能够得到的字典序最小的 QQQ。

思路讲解

OPUS 4.6 的思路

这道题有一个比较干净的做法，不需要动态排序和复杂删除。核心在于把"逆序操作"的想法进一步转化成一个简单的贪心。
关键观察：如何理解最终的 Q
设发布顺序为 π1,π2,…,πn\pi_1, \pi_2, \dots, \pi_nπ1​,π2​,…,πn​。最终的 QQQ 可以这样等价地构造：
按逆序发布顺序（即 πn,πn−1,…,π1\pi_n, \pi_{n-1}, \dots, \pi_1πn​,πn−1​,…,π1​）依次处理每篇博客，对每篇博客从右往左扫描，遇到还没出现过的用户就追加到 QQQ 末尾。
这是因为最后发布的博客决定了 QQQ 最前面的部分（最近被移到前面的用户），倒数第二篇决定了下一段，以此类推。
预处理：有效序列
对每篇博客 iii，从右往左扫描，保留每个用户的首次出现，得到"有效序列" RiR_iRi​。例如博客 [1, 2, 1, 3] 的有效序列为 [3, 1, 2]。
贪心选择
维护一个集合 used 记录已经放入 QQQ 的元素。每一步：

  1. 对每个还未选过的博客，把其有效序列中 used 里的元素跳过，得到"剩余有效序列"。
  2. 选择剩余有效序列字典序最小的博客。
  3. 把它的剩余有效序列追加到 QQQ，并将这些元素加入 used。
    重复直到所有博客选完。
    正确性（交换论证）：如果两个博客 AAA、BBB 的剩余有效序列满足 A′<B′A' < B'A′<B′（字典序），那么先处理 AAA 再处理 BBB，QQQ 的前缀就是 A′A'A′ 开头，一定不劣于 B′B'B′ 开头。当 A′A'A′ 是 B′B'B′ 的前缀时，两种顺序结果相同，不影响。
    用第一个样例验证
    博客有效序列：
  • 博客 1：[6, 4, 3, 2, 1]
  • 博客 2：[1, 5, 2]
  • 博客 3：[3, 2, 9, 1]
    第一步：博客 2 的 [1, 5, 2] 字典序最小（以 1 开头），选它。Q=[1,5,2]Q = [1, 5, 2]Q=[1,5,2]。
    第二步：博客 1 剩余 [6, 4, 3]，博客 3 剩余 [3, 9]。选博客 3（3 < 6）。Q=[1,5,2,3,9]Q = [1, 5, 2, 3, 9]Q=[1,5,2,3,9]。
    第三步：博客 1 剩余 [6, 4]。Q=[1,5,2,3,9,6,4]Q = [1, 5, 2, 3, 9, 6, 4]Q=[1,5,2,3,9,6,4]。与期望答案完全一致。
    复杂度
    时间 O(L2)O(L^2)O(L2)，其中 L=∑li≤3000L = \sum l_i \le 3000L=∑li​≤3000。空间 O(L)O(L)O(L)。只需要一个哈希集合来判断元素是否已使用，不需要任何平衡树、动态排序等复杂结构。
