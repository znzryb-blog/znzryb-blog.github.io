Codeforces Round 1083 (Div. 2)——CF-2205-C. Simons and Posting Blogs

Posted on 2026-03-01 Edited on 2026-03-02

题目大意

题目描述

有 nnn 篇博客，第 iii 篇博客按顺序提到了 lil_ili​ 个用户，用数组 ai=[ai,1,ai,2,…,ai,li]a_i=[a_{i,1},a_{i,2},\dots,a_{i,l_i}]ai​=[ai,1​,ai,2​,…,ai,li​​] 表示。

你需要将这 nnn 篇博客全部发布。维护一个序列 QQQ（初始为空）来记录最近提到的用户列表。你需要选择一个所有 nnn 篇博客的发布顺序进行发布，每次发布一篇博客 iii 时，会对每一个 1≤j≤li1 \le j \le l_i1≤j≤li​ 依次执行以下操作：

  • 如果 ai,ja_{i,j}ai,j​ 已经存在于 QQQ 中，则将 ai,ja_{i,j}ai,j​ 移动到 QQQ 的开头。

  • 否则，将 ai,ja_{i,j}ai,j​ 插入到 QQQ 的开头。

求在发布所有 nnn 篇博客后，所能得到的字典序最小的序列 QQQ。

输入格式

第一行包含一个整数 ttt（1≤t≤10001 \le t \le 10001≤t≤1000），表示测试用例的数量。
每个测试用例第一行包含一个整数 nnn（1≤n≤30001 \le n \le 30001≤n≤3000），表示博客数量。
接下来 nnn 行，每行首先包含一个整数 lil_ili​（1≤li≤30001 \le l_i \le 30001≤li​≤3000），表示第 iii 篇博客提到的用户数量，随后跟着 lil_ili​ 个整数 ai,1,ai,2,…,ai,lia_{i,1},a_{i,2},\dots,a_{i,l_i}ai,1​,ai,2​,…,ai,li​​（1≤ai,j≤1061 \le a_{i,j} \le 10^61≤ai,j​≤106），表示提到的用户列表。
保证所有测试用例中 nnn 的总和不超过 300030003000，∑li\sum l_i∑li​ 的总和不超过 300030003000。

输出格式

对于每个测试用例，输出一行包含若干个整数，表示能够得到的字典序最小的 QQQ。

样例解释

在第一个测试用例中，可以按如下顺序发布博客：

  1. 发布第一篇博客，QQQ 变为 [6,4,3,2,1][6,4,3,2,1][6,4,3,2,1]。

  2. 发布第三篇博客，QQQ 变为 [3,2,9,1,6,4][3,2,9,1,6,4][3,2,9,1,6,4]。

  3. 发布第二篇博客，QQQ 变为 [1,5,2,3,9,6,4][1,5,2,3,9,6,4][1,5,2,3,9,6,4]。

这里存在其他的发布方式，例如：

  1. 发布第三篇博客，QQQ 变为 [3,2,9,1][3,2,9,1][3,2,9,1]。

  2. 发布第一篇博客，QQQ 变为 [6,4,3,2,1,9][6,4,3,2,1,9][6,4,3,2,1,9]。

  3. 发布第二篇博客，QQQ 变为 [1,5,2,6,4,3,9][1,5,2,6,4,3,9][1,5,2,6,4,3,9]。

可以看到 [1,5,2,3,9,6,4][1,5,2,3,9,6,4][1,5,2,3,9,6,4] 的字典序比其他结果更小。如果我们不把第二篇博客放在最后发布，序列的第一个元素将不会是 111。因此 [1,5,2,3,9,6,4][1,5,2,3,9,6,4][1,5,2,3,9,6,4] 就是能得到的字典序最小的数组 QQQ。

在第二个测试用例中，可以按如下顺序发布博客：

  2. 发布第二篇博客，QQQ 保持自身为 [6,1][6,1][6,1]。
