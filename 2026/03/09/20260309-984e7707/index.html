Codeforces Round 1085 (Div. 1 + Div. 2)——CF-2207-B. One Night At Freddy's（一定要关注数据范围限制）

Posted on 2026-03-09

题目大意

题目描述

一共有 mmm 个玩偶，初始危险值 d1,d2,…,dmd_1, d_2, \dots, d_md1​,d2​,…,dm​ 均为 0。

整个过程持续 ℓ\ellℓ 秒。在每一秒内，会有一个（且仅有一个）玩偶的危险值增加 1。你可以时刻观察到所有玩偶当前的危险值。

你有 nnn 次使用手电筒的机会，分别在固定的时间点 a1,a2,…,ana_1, a_2, \dots, a_na1​,a2​,…,an​（1≤a1<a2<⋯<an≤ℓ1 \le a_1 < a_2 < \dots < a_n \le \ell1≤a1​<a2​<⋯<an​≤ℓ）秒后发生。
每次使用手电筒时，你可以选择恰好一个玩偶，将其危险值重置为 0。每次的选择是互相独立的。

最终的“总危险值”定义为 ℓ\ellℓ 秒结束时，所有玩偶危险值的最大值，即 max⁡1≤j≤mdj\max_{1 \le j \le m} d_jmax1≤j≤m​dj​。
你需要求出一个最小的整数 xxx，使得无论每一秒是哪个玩偶的危险值增加，你都能通过合理地使用手电筒，保证最终的总危险值不超过 xxx。

输入格式

第一行包含一个整数 ttt（1≤t≤1041 \le t \le 10^41≤t≤104），表示测试用例的数量。

每个测试用例的第一行包含三个整数 n,m,ℓn, m, \elln,m,ℓ（1≤n,m,ℓ≤2⋅1051 \le n, m, \ell \le 2 \cdot 10^51≤n,m,ℓ≤2⋅105，n≤ℓn \le \elln≤ℓ，1≤m⋅ℓ≤2⋅1051 \le m \cdot \ell \le 2 \cdot 10^51≤m⋅ℓ≤2⋅105），分别代表手电筒使用次数、玩偶数量以及总时长。

第二行包含 nnn 个整数 a1,a2,…,ana_1, a_2, \dots, a_na1​,a2​,…,an​（1≤a1<a2<⋯<an≤ℓ1 \le a_1 < a_2 < \dots < a_n \le \ell1≤a1​<a2​<⋯<an​≤ℓ），代表可以使用手电筒的时间点。

保证所有测试用例中 m⋅ℓm \cdot \ellm⋅ℓ 的总和不超过 2⋅1052 \cdot 10^52⋅105。

输出格式

对于每个测试用例，输出一个整数，即保证最终总危险值不超过的最小值 xxx。

样例解释

在第一个测试用例中，有 222 个玩偶，总时长为 101010 秒，在第 101010 秒后可使用一次手电筒。可以证明 x=5x=5x=5 总是可行的：101010 秒后，必定有一个玩偶的危险值至少为 555，另一个至多为 555。我们对着危险值较大的那个使用手电筒将其清零，那么最终的最大危险值至多为 555。可以证明 555 是 xxx 的最小可能值。

在第二个测试用例中，只有 111 个玩偶，总时长为 323232 秒。由于只有 111 个玩偶，它的危险值每秒必定增加 111。在最后一次使用手电筒（第 252525 秒）时，我们将它的危险值清零。在这之后距离结束还有 777 秒，因此最终危险值必定为 777。

在第三个测试用例中，可以证明最小可能的值 xxx 为 191919。

思路讲解

就是这个 B，哎，感觉基本上是想到了，但是没关注到一个关键的这个数据范围限制。

一定要关注数据范围限制。

你看到这个，基本就知道简单的贪心和这个二分答案是解决不了的。

其实这道题目，这个思路还是不难的，就是，我们想让我们的每一次操作都在最终的答案中贡献最大，是不是？

然后我们发现，如果说后面还有 N 个手电筒，我们如果给 N 个球加，我们其实对这个最终答案的贡献是 0。

那怎么办？我们可以给 N+1 个球加。
