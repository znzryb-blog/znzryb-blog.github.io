题目描述

给定一个包含 nnn 张牌的牌堆，每张牌有点数 aia_iai​。有 mmm 个玩家依次循环摸牌（即玩家 111 摸第 111 张，玩家 222 摸第 222 张，……，玩家 mmm 摸第 mmm 张，然后玩家 111 再摸第 m+1m+1m+1 张，以此类推），直到牌摸完。
玩家 kkk 可以在摸牌前进行最多一次操作：将牌堆从任意位置切开，交换上下两部分。这等价于对数组进行任意次数的循环移位。
求玩家 kkk 能摸到的牌的点数总和的最大值。

解题思路

首先分析玩家 kkk 摸到的牌的数量和位置。
由于牌是按照 111 到 mmm 循环分发的，在洗牌（循环移位）后的新牌堆中，玩家 kkk 必定摸走下标为 k,k+m,k+2m,…k, k+m, k+2m, \dotsk,k+m,k+2m,… 的牌（下标从 111 开始）。
因为牌堆总数为 nnn，所以玩家 kkk 摸到的牌的数量始终为定值 C=⌊n−km⌋+1C = \lfloor \frac{n-k}{m} \rfloor + 1C=⌊mn−k​⌋+1。
无论我们将原牌堆循环移位多少次，新牌堆中下标相邻为 mmm 的位置，在原牌堆中它们的下标差值也必定是 m(modn)m \pmod nm(modn)。
因此，玩家 kkk 摸到的 CCC 张牌，在原数组中恰好对应了一条从某个起点开始、每次下标加上 m(modn)m \pmod nm(modn)、长度为 CCC 的路径。
这提示我们将原数组的下标视为一个图，每个下标 iii 向 (i+m) mod n(i+m) \bmod n(i+m)modn 连边。
根据数论性质，这个图由 g=gcd⁡(n,m)g = \gcd(n, m)g=gcd(n,m) 个不相交的环构成，每个环的长度为 L=ngL = \frac{n}{g}L=gn​。
由于循环移位的偏移量 sss 可以取 000 到 n−1n-1n−1 中的任意整数，路径的起点可以取遍所有 nnn 个位置。
所以，问题转化为：在这 ggg 个环中，寻找所有长度恰好为 CCC 的连续子段，求出其中点数总和的最大值。
算法步骤如下：

  1. 计算环的个数 g=gcd⁡(n,m)g = \gcd(n, m)g=gcd(n,m)，环长 L=ngL = \frac{n}{g}L=gn​，以及子段长度 C=⌊n−km⌋+1C = \lfloor \frac{n-k}{m} \rfloor + 1C=⌊mn−k​⌋+1。
  2. 枚举 i∈[0,g−1]i \in [0, g-1]i∈[0,g−1]，遍历这 ggg 个环。
  3. 对于每个环，按 i,(i+m) mod n,(i+2m) mod n,…i, (i+m) \bmod n, (i+2m) \bmod n, \dotsi,(i+m)modn,(i+2m)modn,… 的顺序提取出环上的 LLL 个元素。
  4. 在提取出的环上，使用滑动窗口算法求出长度为 CCC 的最大连续子段和。注意环的连续性，滑动窗口在末尾时需要绕回开头。
  5. 在所有环的所有长度为 CCC 的子段和中取最大值即为答案。
    经过证明可以发现，窗口长度 CCC 始终满足 C≤LC \le LC≤L，因此滑动窗口的长度绝不会超出环的单圈长度。

我们可以采用这道题目的一个思路，就是，截止时间，ddl 非常难以处理，但是反过来看，就是这个激活时间，激活时间以后，这个点就想选就选，不想选就不选了。

然后，我们再来看这个东西，就是我们已经想到了这个 ddl 就是这个激活时间，然后我们现在的问题就是每个点的这个 ddl 是什么，能不能批量处理呢？

我们来看最差情况下和最好情况下的这个差别：最坏情况下（也就是假设所有的我们自己选的都在前面），第 kkk 次操作的剩余数字的下标要求是 i>k(x+1)i>k(x+1)i>k(x+1) 。在最好情况下，第 kkk 次操作的剩余数字的下标要求是 i>kxi>kxi>kx。那么 (kx,k(x+1))(kx,k(x+1))(kx,k(x+1)) 中的这个数字的 ddl 是 k+1k+1k+1，即在 k+1k+1k+1 次操作之前，这个数都是一定可以选的。((k+1)x,(k+1)(x+1))((k+1)x,(k+1)(x+1))((k+1)x,(k+1)(x+1))。

通过上面👆的这个分析，我们会发现 ddl 如果采用真实游戏结束回合的定义，也就是什么时候会被取走，那么这个点的 ddl 就不是固定的，这非常不适合我们进行优化以及操作。

算法思路

本题可以使用动态规划来解决。
为了避免重复计算本质相同的子序列，我们可以采用贪心匹配的思想：对于任何一个合法的子序列，为了给后续字符留下最大的选择空间，我们总是贪心地在原串中寻找它的第一个可用匹配位置。在这个约束下，每个合法的 kkk-松散子序列都唯一对应原串中的一种提取方式。
我们定义 VxV_xVx​ 为：在后缀 Sx…nS_{x \dots n}Sx…n​ 中，能够选出的本质不同的合法序列数量（第一步选择没有 +k+k+k 的位置限制，但后续的选择仍需满足距离 >k>k>k 的限制）。
考虑如何从 Vx+1V_{x+1}Vx+1​ 转移到 VxV_xVx​：
与后缀 Sx+1…nS_{x+1 \dots n}Sx+1…n​ 相比，后缀 Sx…nS_{x \dots n}Sx…n​ 多了一个字符 SxS_xSx​ 可以作为序列的开头。

  1. 如果我们将 SxS_xSx​ 作为子序列的第一个字符，那么下一个字符必须从下标 ≥x+k+1\ge x+k+1≥x+k+1 的位置开始选择。这种情况能产生的合法子序列数量为 1+Vx+k+11 + V_{x+k+1}1+Vx+k+1​（其中 111 代表仅由 SxS_xSx​ 构成的长度为 111 的子序列）。
  2. 然而，在 Vx+1V_{x+1}Vx+1​ 的结果中，可能已经包含了以 SxS_xSx​ 这个字符开头的序列——它们是把 SxS_xSx​ 的下一次出现位置（记为 nxtxnxt_xnxtx​）作为起点的。为了防止重复计算以该字符开头的序列，我们需要把这部分重复的序列减去。以 nxtxnxt_xnxtx​ 作为起点能产生的子序列数量为 1+Vnxtx+k+11 + V_{nxt_x+k+1}1+Vnxtx​+k+1​。
    综上所述，我们可以得出状态转移方程。
    定义辅助数组 WxW_xWx​ 表示选择位置 xxx 作为开头时的组合数：
    如果 x≤nx \le nx≤n，则 Wx←1+Vx+k+1W_x \leftarrow 1 + V_{x+k+1}Wx​←1+Vx+k+1​；否则 Wx←0W_x \leftarrow 0Wx​←0。
    那么状态转移方程为：
    Vx←(Vx+1+Wx−Wnxtx(mod998244353)+998244353) mod 998244353V_x \leftarrow (V_{x+1} + W_x - W_{nxt_x} \pmod{998244353} + 998244353) \bmod 998244353Vx​←(Vx+1​+Wx​−Wnxtx​​(mod998244353)+998244353)mod998244353
    我们只需从右向左（xxx 从 nnn 递减到 111）遍历字符串计算 VxV_xVx​，最终的答案即为 V1V_1V1​。

复杂度分析与边界情况

算法说明：
我们利用了子序列自动机的贪心匹配性质，将“本质不同”的限制转换为“每次必须选取该字符第一次出现的位置”。接着使用动态规划，通过类似于经典本质不同子序列计数（即 k=0k=0k=0 时）的加减容斥思想，巧妙避开了状态中带有位置距离限制的遍历，将每一次状态转移做到了 O(1)O(1)O(1) 的极致效率。
时间复杂度：

  1. 预处理 nxt 数组需要逆序遍历一次字符串，花费 O(n)O(n)O(n) 的时间。
  2. 动态规划过程只需要从后向前遍历一次，每次循环内部都是简单的条件判断和常数级别的加减运算，花费 O(n)O(n)O(n) 的时间。
    因此对于每组测试用例，时间复杂度为 O(n)O(n)O(n)。所有测试用例的总时间复杂度为 O(∑n)O(\sum n)O(∑n)，最大计算量约 10610^6106 次操作，可以在 111 秒的时间限制内极其轻松地通过。
    空间复杂度：
    我们创建了长度为 n+1n+1n+1 的 nxt 数组，以及长度为 n+k+5n+k+5n+k+5 的动态规划数组 V。两个数组的大小均为 O(n)O(n)O(n) 级别。
    由于每次都是动态分配，空间复杂度为 O(n)O(n)O(n)。最大极限情况下占用内存不足 15MB15\text{MB}15MB，严格符合并远低于 512MB512\text{MB}512MB 的内存限制要求。
    边界情况处理：
  3. k≥nk \ge nk≥n：此时任何选取的字符下标加上 k+1k+1k+1 都会超出字符串长度，即 Vi+k+1V_{i+k+1}Vi+k+1​ 必然为 000。此时原串中每种出现过的字符只会被单独算作一个长度为 111 的子序列（由于 WnxtxW_{nxt_x}Wnxtx​​ 的减法抵消作用，每个不同的字母只会全局计算一次）。代码中分配的空间 n + k + 5 完美覆盖了这一情况，不仅逻辑正确也不会出现数组越界。
  4. 取模运算的负数情况：由于状态转移中含有减法（容斥），直接取模可能会导致负数结果传递。代码中采用了加减法与判断结合的方式（v -= w_nxt; if (v < 0) v += MOD;）手动修正负数。这种方式不仅避免了除法取模运算符 % 带来的指令周期损耗，还确保了最终状态值绝对正确且总是正整数。
